# New use case

Generate a **new WordPress plugin** under `plugins/` that registers a custom CoverKit use case.

Follow every step in [`.cursor/rules/new-usecase.mdc`](../rules/new-usecase.mdc).

User passes a short slug or name after the command (e.g. `/new-usecase email-header` or `/new-usecase Pinterest board`). If missing, ask once, then continue.
